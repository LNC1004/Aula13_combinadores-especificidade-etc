Esplicação tags e propriedades novas

Regra da Cascata

A regra da cascata é responsável por determinar qual estilo será aplicado a um elemento HTML quando houver vários outros iguais a ele. É um dos conceitos mais fundamentais e, ao mesmo tempo, mais difíceis para ser dominado por iniciantes.

A cascata segue três princípios principais que agem como sitema de pontuação:

  1. Ordem e Origem: De onde vem o estilo?
  2. Especificidade: Quão específico é o seletor?
  3. Ordem de Código: Qual foi a última regra a ser declarada?

Especificidade

É a forma como o navegador decide quais regras de CSS aplicar a um elemento.

A especificidade é feita em forma de pontos e é diferenciada pelos seletores:

  1. Seletor pela tag: Tem a menor pontuação (0,0,1). Aponta todos os elementos de um tipo específico.
  2. Seletor pela classe: Tem uma pontuação média (0,1,0). Aponta um grupo de elementos que compartilham a mesma classe.
  3. Seletor pelo id: Tem a maior pontuação (1,0,0). Aponta um único elemento, já que os IDs são únicos.

Combinadores

Combinadores são símbolos ou palavras-chave que combinam seletores para definir uma relação entre eles.

Existem três tipos principais de combinadores:

  1. Combinador de espaço (.primeira h1): Seleciona todos os h1s descendentes de um elemento com a classe .primeira
  2. Combinador Filho (.primeira > h1): Seleciona somente um h1 que seja um "filho" direto da classe .primeira
  3. Combinador de soma (.primeira h1 + h1): Seleciona um h1 que seja um "irmão" do primeiro h1

Pseudo-classes

Pseudo-classes são palavras-chave especiais no CSS que permitem selecionar um elemento com base em seu estado.

Pseudo-classes principais:

  • first-child: Seleciona o primeiro elemento filho de um elemento pai
  • last-child: Seleciona o último elemento filho de um elemento pai
  • nth-child(n): Seleciona o enésimo elemento filho, onde n pode ser um número, palavra-chave ou fómula
  • nth-of-type(n): Selecion um elemento entre todos os elementos irmãos

Display

A propriedade display define o tipo de caixa de layout de um elemento.

Os valores mais comuns são:

  • inline: Ocupa apenas o espaço necessário, não inicia em outra linha, ignora width e height, aceitam margin e padding somente nas laterais
  • block: Ocupa 100% da largura disponível, sempre inicia em uma nova linha, aceita width, height, margin e padding em qualquer direção

display-inline usa por padrão: a, span, em, strong, img, input, label, button.

Já o display-block usa por padrão: div, h1 a h6, p, ol, ul, li, header, footer, section, article, table.

Serventia das propriedades de tamanho

Uma página é formada como uma caixa, e possuí partes relevantes ou não.

São essas:

  1. padding: É o espaço entre o conteúdo e a borda (Preenchimento), tem estilo bróprio e a maioria das cores
  2. border: É a linha que envolve o padding (Borda da página). Contém a cor de fundo
  3. margin: É o espaço externo que separa um elemento de outro (Margem). É transparente

Shorthand e sufixos

São usados para alterar tamanho

Shorthads

Usam px (pixels) como medida: 10px, 20px, 30px, etc.

Sufixos

Define a posição para onde será alterado o tamanho: top(em cima), bottom (em baixo), left (esquerda), right (direita).

Box-sizing

Define o tamanho de um elemento.

Usa principalmente o border-box.

Com o border-box, a largura e a altura que você define incluem o preenchimento e a borda. A margem, no entanto, continua a ser calculada separadamente. Então, se você definir a largura do mesmo div para 200px e adicionar 20px de preenchimento e 5px de borda, a largura total do elemento na página ainda será 200px. O preenchimento e a borda são "encaixados" dentro da dimensão total, e a área de conteúdo será reduzida para acomodá-los.
